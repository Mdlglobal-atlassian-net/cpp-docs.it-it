# Errore del compilatore C2244
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

'identificatore': impossibile trovare una corrispondenza tra la definizione della funzione e una dichiarazione esistente  
  
 L'operatore unario \+ è stato utilizzato in modo insolito prima di una chiamata di funzione senza parentesi.  
  
 Questo errore si verifica solo nei progetti C\+\+.  
  
 Il seguente codice di esempio genera l'errore C2244:  
  
```  
// C2244.cpp  
int func(char) {  
   return 0;  
}   
  
int func(int) {  
   return 0;  
}  
  
int main() {  
   +func;   // C2244  
}  
```  
  
 L'errore C2244 può inoltre verificarsi quando si utilizza una firma della funzione non corretta per una funzione membro di un modello di classe.  
  
```  
// C2244b.cpp  
// compile with: /c  
template<class T>   
class XYZ {  
   void func(T t);  
};  
  
template<class T>  
void XYZ<T>::func(int i) {}   // C2244 wrong function signature  
// try the following line instead  
// void XYZ<T>::func(T t) {}  
```  
  
 L'errore C2244 può inoltre verificarsi quando si utilizza una firma della funzione non corretta per un modello di funzioni membro.  
  
```  
// C2244c.cpp  
// compile with: /c  
class ABC {  
   template<class T>   
   void func(int i, T t);  
};  
  
template<class T>  
void ABC::func(int i) {}   // C2244 wrong signature  
// try the following line instead  
// void ABC::func(int i, T t) {}  
```  
  
 Non è possibile specializzare parzialmente un modello di funzione.  
  
```  
// C2244d.cpp  
template<class T, class U>  
class QRS {  
   void func(T t, U u);  
};  
  
template<class T>  
void QRS<T,int>::func(T t, int u) {}   // C2244  
```
