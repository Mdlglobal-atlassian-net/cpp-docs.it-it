# <a name="compiler-warning-level-2-c5046"></a>Del compilatore (livello 2) Avviso C5046

> «*funzione*': Tipo che interessa di simboli con collegamento interno non definito

## <a name="remarks"></a>Note

Il compilatore ha rilevato un utilizzo di una funzione che non dispone di una definizione, ma la firma di questa funzione prevede tipi che non sono visibili all'esterno di questa unità di conversione. Poiché questi tipi non visibili esternamente, nessun altra unità di conversione può fornire una definizione per questa funzione, in modo che il programma non può essere collegato.

I tipi che non sono visibili tra unità di conversione includono:

- I tipi dichiarati all'interno di uno spazio dei nomi anonimo

- Classi senza nome o locale

- Specializzazioni dei modelli che usano questi tipi come argomenti di modello.

Questo avviso è stata introdotta in Visual Studio 2017 versione 15.8.

## <a name="example"></a>Esempio

In questo esempio mostra due C5046 avvisi:

```cpp
// C5046p.cpp
// compile with: cl /c /W2 C5046p.cpp

namespace {
    struct S {
        // S::f is inside an anonymous namespace and cannot be defined outside
        // of this file. If used, it must be defined somewhere in this file.
        int f();
    };
}

// g has a pointer to an unnamed struct as a parameter type. This type is
// distinct from any similar type declared in other files, so this function
// cannot be defined in any other file.
// Note that if the struct was declared "typedef struct { int x; int y; } S, *PS;"
// it would have a "typedef name for linkage purposes" and g could be defined
// in another file that provides a compatible definition for S.

typedef struct { int x; int y; } *PS;
int g(PS p);

int main()
{
    S s;
    s.f();      // C5046 f is undefined and can't be defined in another file.
    g(nullptr); // C5046 g is undefined and can't be defined in another file.
}
```

Per risolvere questi problemi, definire le funzioni in questo file.
