# Avviso del compilatore (livello 1) C4929
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

'file': la libreria dei tipi contiene un elemento union: verrà ignorato il qualificatore 'embedded\_idl'  
  
 Impossibile applicare l'attributo embedded\_idl di [\#import](../../preprocessor/hash-import-directive-cpp.md) alla libreria dei tipi perché quest'ultima contiene un elemento union.  Per risolvere il problema, non utilizzare l'attributo embedded\_idl.  
  
## Esempio  
 Nell'esempio riportato di seguito viene definito un componente.  
  
```  
// C4929a.cpp  
// compile with: /LD /link /TLBOUT:C4929a.tlb  
#include <objbase.h>  
[module(name="Test")];  
[public, switch_type(short)] typedef union _TD_UNION_TYPE   {  
   [case(24)]  
      float fM;  
   [case(25)]  
      double dMN;  
   [default]  
      int x;  
} TD_UNION_TYPE;  
  
[export, public] typedef struct _TDW_TYPE {  
   [switch_is(sU)] TD_UNION_TYPE w;  
      short sU;  
} TD_TYPE;  
  
[object, uuid("00000000-0000-0000-0000-000000000001")]  
__interface I {  
   HRESULT f(TD_TYPE*);  
};  
  
[coclass, uuid("00000000-0000-0000-0000-000000000002")]  
struct C : I {  
   HRESULT f(TD_TYPE*) { return 0; }  
};  
```  
  
## Esempio  
 Nell'esempio seguente viene generato l'errore C4929:  
  
```  
// C4929b.cpp  
// compile with: /c /W1  
#import "C4929a.tlb" embedded_idl   // C4929  
```
