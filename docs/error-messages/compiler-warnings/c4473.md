# <a name="compiler-warning-level-1-c4473"></a>Compilatore Warning (level 1) C4473

> '*funzione*': passati argomenti insufficienti per segnaposto della stringa di formato e i relativi parametri prevedono *numero* argomenti variadic, ma *numero* sono stati specificati i manca l'argomento variadic *indice* richiesto da stringa di formato '*format_string*' in questo argomento viene usato da questo argomento viene usato come larghezza del campo in questo argomento viene usato un identificatore di conversione come una precisione di questo argomento viene usato come dimensioni del buffer

Il compilatore ha rilevato una mancata corrispondenza tra il numero di argomenti necessari per soddisfare i segnaposto in una stringa di formato e il numero di argomenti forniti. Utilizzo corretto delle famiglie di funzioni printf e scanf delle funzioni variadic, è necessario fornire un numero di argomenti come specificato dalla stringa di formato. Alcuni segnaposto richiedono argomenti aggiuntivi, per specificare la larghezza, precisione o le dimensioni del buffer, nonché un argomento per il contenuto. Una mancata corrispondenza indica in genere che è presente un bug nel codice.

Per informazioni sugli argomenti associati ai segnaposto della stringa di formato della famiglia printf, vedere [sintassi per la specifica di formato: funzioni printf e wprintf](../../c-runtime-library/format-specification-syntax-printf-and-wprintf-functions.md). Vedere la documentazione per informazioni specifiche per la funzione *funzione*.

Questo avviso è stata introdotta in Visual Studio 2015.

## <a name="example"></a>Esempio

Le due modalità avviso C4473 può essere causato da una stringa di formato seguenti mostrano esempi di mancata corrispondenza con i relativi argomenti e viene mostrato inoltre come risolvere i problemi.

```cpp
// c4473p.cpp
// compile with: cl /c /W4 c4473p.cpp
#include <stdio.h>

void print_func(int a, float f)
{
    printf_s("This takes %*f arguments.\n", f); // C4473, missing * width argument
    printf_s("This takes %*f arguments.\n", a, f); // fix: supply width argument
}
```

Nel primo messaggio di errore, il compilatore rileva che un argomento non è presente, ma non è possibile stabilire se l'argomento mancante è la larghezza o del contenuto, in modo che si presuppone che l'argomento fornito è il primo, per la larghezza e l'argomento mancante è il secondo , per l'identificatore di conversione del tipo. È necessario esaminare la stringa di formato per determinare l'effettiva argomento mancante.

```cpp
// c4473s.cpp
// compile with: cl /c /W4 c4473s.cpp
#include <stdio.h>

void scan_func(int a, float f)
{
    char name[20];

    scanf_s("%s", &name);                // C4473, missing buffer size argument
    scanf_s("%s", &name, sizeof(name));  // fix: supply buffer size argument
}
```

In questo esempio **scanf_s** richiede due argomenti per ogni segnaposto, uno per fornire l'indirizzo in cui scrivere e il secondo per specificare la dimensione del primo. Consultare la documentazione per ogni funzione di libreria per una spiegazione degli argomenti obbligatori.
