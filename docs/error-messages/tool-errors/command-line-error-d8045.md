# Errore della riga di comando D8045
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

impossibile compilare il file C 'file' con l'opzione \/clr  
  
 È possibile passare solo file di codice sorgente C\+\+ a una compilazione eseguita con l'opzione **\/clr**.  Utilizzare **\/TP** per compilare un file c come file cpp. Per ulteriori informazioni, vedere [\/Tc, \/Tp, \/TC, \/TP \(Specifica il tipo di file di origine\)](../../build/reference/tc-tp-tc-tp-specify-source-file-type.md).  
  
 Per ulteriori informazioni, vedere [\/clr \(Compilazione Common Language Runtime\)](../../build/reference/clr-common-language-runtime-compilation.md).  
  
 L'errore D8045 può anche verificarsi se si compila un'applicazione ATL mediante Visual C\+\+.  Per ulteriori informazioni, vedere [Procedura: eseguire la migrazione a \/clr](../../dotnet/how-to-migrate-to-clr.md).
