# Nozioni fondamentali sul controllo composito di ATL
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Un controllo composito è un tipo di controllo ActiveX che può contenere \(simile a una finestra di dialogo\) altri controlli ActiveX o controlli Windows.  Una volta che il controllo composito viene compilato, può essere inserito in un controllo ActiveX può essere ospitato.  
  
 La creazione guidata progetto ATL e la finestra di dialogo **Aggiungi classe** automatizzano il processo di creazione e distribuzione di un progetto di controllo composito, simile al risultato di eseguire la procedura guidata per creare un framework applicazione MFC.  Il processo di sviluppo è costituita da cinque passaggi:  
  
-   [Creare un progetto ATL](../atl/reference/creating-an-atl-project.md)  
  
-   [Inserimento del controllo composito](../atl/inserting-a-composite-control.md)  
  
-   [Modificare il progetto ATL](../atl/modifying-the-atl-project.md)  
  
-   [Aggiunta di funzionalità al controllo composito](../atl/adding-functionality-to-the-composite-control.md)  
  
-   [Compilazione e test al progetto ATL](../atl/building-and-testing-the-atl-project.md)
