# <a name="ftellnolock-ftelli64nolock"></a>_ftell_nolock, _ftelli64_nolock
Ottiene la posizione corrente di un puntatore del file, senza bloccare il thread.  
  
## <a name="syntax"></a>Sintassi  
  
```  
long _ftell_nolock(   
   FILE *stream   
);  
__int64 _ftelli64_nolock(   
   FILE *stream   
);  
```  
  
#### <a name="parameters"></a>Parametri  
 `stream`  
 Struttura `FILE` di destinazione.  
  
## <a name="return-value"></a>Valore restituito  
 Uguale a `ftell` e `_ftelli64`. Per altre informazioni, vedere [ftell, _ftelli64](../../c-runtime-library/reference/ftell-ftelli64.md)**.**  
  
## <a name="remarks"></a>Note  
 Queste funzioni sono le versioni che non bloccano thread rispettivamente di `ftell` e `_ftelli64`. Sono identici a `ftell` e `_ftelli64` ad eccezione del fatto che non sono protette da interferenze da altri thread. Queste funzioni potrebbero essere più veloci perché non comportano un sovraccarico che blocca altri thread. Utilizzare queste funzioni solo in contesti thread-safe come applicazioni a thread singolo o dove l'ambito chiamante già gestisce l'isolamento del thread.
