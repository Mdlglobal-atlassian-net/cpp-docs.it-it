# ODBC: configurazione di un&#39;origine dati ODBC
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Per utilizzare un'[origine dati](../../data/odbc/data-source-odbc.md) con un'applicazione sviluppata, Ã¨ necessario utilizzare l'Amministratore ODBC per configurarla.  L'Amministratore ODBC tiene traccia delle origini dati disponibili e delle relative informazioni di connessione nel Registro di sistema di Windows.  Utilizzare l'Amministratore ODBC per aggiungere, modificare ed eliminare le origini dati nell'apposita finestra di dialogo e per aggiungere ed eliminare i driver ODBC.  
  
> [!NOTE]
>  Queste informazioni sono valide quando si utilizzano le classi DAO MFC per accedere a ODBC e quando si utilizzano le classi ODBC MFC.  
  
 L'Amministratore ODBC viene installato automaticamente con il supporto per il database della libreria MFC.  Per ulteriori informazioni sull'Amministratore ODBC, vedere [Amministratore ODBC](../../data/odbc/odbc-administrator.md) e il sistema di Guida di riferimento per l'API ODBC.  
  
 Per informazioni sulla scrittura di programmi di amministrazione e installazione ODBC per le applicazioni di database MFC, vedere la[Nota tecnica 48](../../mfc/tn048-writing-odbc-setup-and-administration-programs.md).
