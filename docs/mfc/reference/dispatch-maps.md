# Mappe di invio
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

L'automazione OLE fornisce modi per chiamare metodi e accedere alle proprietà nelle applicazioni.  Il meccanismo fornito dalla libreria MFC per inviare le quali è "la mappa submit", che definisce i nomi interni ed esterni di funzioni e delle proprietà di un oggetto nonché i tipi di dati delle proprietà stessi e gli argomenti della funzione.  
  
### Mappe di invio  
  
|||  
|-|-|  
|[DECLARE\_DISPATCH\_MAP](../Topic/DECLARE_DISPATCH_MAP.md)|Indica che una mappa di divisione verrà utilizzata per esporre i metodi e le proprietà della classe \(deve essere utilizzato nella dichiarazione di classe\).|  
|[BEGIN\_DISPATCH\_MAP](../Topic/BEGIN_DISPATCH_MAP.md)|Iniziare la definizione di una mappa di invio.|  
|[END\_DISPATCH\_MAP](../Topic/END_DISPATCH_MAP.md)|Termina la definizione di una mappa di invio.|  
|[DISP\_FUNCTION](../Topic/DISP_FUNCTION.md)|Utilizzato in una mappa di invio per definire una funzione di automazione OLE.|  
|[DISP\_PROPERTY](../Topic/DISP_PROPERTY.md)|Definisce una proprietà di automazione OLE.|  
|[DISP\_PROPERTY\_EX](../Topic/DISP_PROPERTY_EX.md)|Definisce una proprietà di automazione OLE e assegnare delle funzioni get e set.|  
|[DISP\_PROPERTY\_NOTIFY](../Topic/DISP_PROPERTY_NOTIFY.md)|Definisce una proprietà di automazione OLE con la notifica.|  
|[DISP\_PROPERTY\_PARAM](../Topic/DISP_PROPERTY_PARAM.md)|Definisce una proprietà di automazione OLE che accetta parametri e i nomi delle funzioni get e set.|  
|[DISP\_DEFVALUE](../Topic/DISP_DEFVALUE.md)|Rende a una proprietà esistente il valore predefinito di un oggetto.|
