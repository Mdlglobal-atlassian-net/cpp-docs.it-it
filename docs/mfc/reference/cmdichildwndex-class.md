# <a name="cmdichildwndex-class"></a>CMDIChildWndEx (classe)

La `CMDIChildWndEx` classe fornisce la funzionalità di una finestra figlio di interfaccia a documenti multipli (MDI) di Windows. Estende la funzionalità della [classe CMDIChildWnd](../../mfc/reference/cmdichildwnd-class.md). Il framework richiede questa classe quando un'applicazione MDI utilizza determinate classi MFC.

Per ulteriori dettagli, vedere il codice sorgente che si trova nella cartella **\\\\mfc di VC atlmfc\\** dell'installazione di Visual Studio.

## <a name="syntax"></a>Sintassi

```
class CMDIChildWndEx : public CMDIChildWnd
```

## <a name="cmdichildwndexdockpaneleftof"></a><a name="dockpaneleftof"></a>CMDIChildWndEx::DockPaneLeftOf

```

    CPane* pBar,
    CPane* pLeftOf);
```

### <a name="parameters"></a>Parametri

*pBar (Barra)*<br/>
Puntatore al riquadro da ancorare.

*pLeftOf*<br/>
Puntatore al riquadro che funge da punto di riferimento.

Chiamare questo metodo quando si desidera ancorare più riquadri in ordine predefinito.

### <a name="return-value"></a>Valore restituito

TRUESe il metodo ha esito positivo. in caso contrario, FALSE.

Un documento è ciò che viene visualizzata la finestra figlio MDI. In genere, nella finestra vengono visualizzati i dati caricati o salvati in un file. Pertanto, il nome del documento è il nome del file. L'implementazione `GetDocumentName` predefinita di `CDocument::GetPathName`restituisce una stringa ottenuta da .

Se nella finestra viene visualizzato un documento che non viene caricato da un file, eseguire l'override di questo metodo in una classe derivata e restituire un identificatore di documento univoco.

`GetDocumentName`viene chiamato dal framework quando salva lo stato di tutti i documenti aperti. La stringa restituita viene scritta nel Registro di sistema.

Quando il framework ripristina lo stato in un secondo momento, il nome del documento viene letto dal Registro di sistema e passato a [CMDIFrameWndEx::CreateDocumentWindow](../../mfc/reference/cmdiframewndex-class.md#createdocumentwindow). Eseguire l'override di questo metodo in una classe derivata da [CMDIFrameWndEx](../../mfc/reference/cmdiframewndex-class.md)e creare o aprire un documento con questo nome e letto nel file con questo nome. Se il documento non è basato su un file, creare il documento in base all'identificatore del documento stesso. È consigliabile eseguire le azioni precedenti solo se si intende salvare e ripristinare i documenti.
