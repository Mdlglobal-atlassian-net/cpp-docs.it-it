# Utilizzo delle barre degli strumenti precedenti
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

Se sono state utilizzate versioni precedenti di Visual C\+\+ per creare barre degli strumenti personalizzate, la nuova implementazione della classe [CToolBar](../mfc/reference/ctoolbar-class.md) potrebbe provocare problemi.  La vecchia implementazione è ancora supportata, così da non dover rinunciare alla propria vecchia barra degli strumenti per utilizzare la nuova funzionalità.  
  
 L'esempio DOCKTOOL non utilizza barre degli strumenti con il vecchio stile, solo barre degli strumenti con lo stile nuovo.  
  
 Non è possibile modificare barre degli strumenti con lo stile vecchio con l'editor di risorse della barra degli strumenti.
