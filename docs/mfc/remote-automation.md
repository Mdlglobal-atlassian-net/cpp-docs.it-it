# <a name="remote-automation"></a>Automazione remota
> [!NOTE]
>  Automazione remota è una tecnologia obsoleta e non deve essere utilizzato per le nuove applicazioni. Visual C++ non supporta Windows 95. Casi in cui è necessario supportare l'automazione remota sono descritti in [dove Does remoto automazione adatta In](where-does-remote-automation-fit-in-q.md).  
  
 Automazione remota è un tipo di [automazione](../mfc/automation.md) che consente a un consumer di interfaccia per l'esecuzione di un provider di interfaccia che risiede in un altro computer, ad esempio, in una rete.  
  
 In questo articolo viene illustrato come creare oggetti di automazione che possono essere richiamati ed eseguiti in modalità remota e come creare controller di automazione che possono utilizzare questi oggetti di automazione remota. Inoltre vengono esaminate le opzioni di configurazione e vengono illustrate le differenze principali tra l'automazione remota e DCOM (la versione distribuita di COM e OLE che consente alle interfacce diverse da quelle correlate all'automazione di essere richiamate ed eseguite in modalità remota).
