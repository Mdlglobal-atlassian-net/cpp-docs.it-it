# <a name="sequence-of-operations-for-building-mfc-applications"></a>Sequenza delle operazioni per la compilazione di applicazioni MFC

La tabella seguente illustra la sequenza generale che potrebbe seguono in genere quando si sviluppa l'applicazione MFC.

### <a name="sequence-for-building-an-application-with-the-framework"></a>Sequenza per la creazione di un'applicazione con il Framework

|Attività|Davvero|Il framework esegue|
|----------|------------|------------------------|
|Creare una struttura applicazione.|Eseguire la [Creazione guidata applicazione MFC](../mfc/reference/mfc-application-wizard.md). Specificare le opzioni desiderate nelle pagine delle opzioni. Le opzioni includono rendere l'applicazione, un componente COM, contenitore o entrambe le cose; aggiunta di automazione; e creazione di database compatibili con l'applicazione.|La creazione guidata applicazione MFC crea i file per una struttura applicazione, inclusi i file di origine per l'applicazione, documenti, visualizzazione e finestre cornice; un file di risorse. un file di progetto. e altri, tutti adattati in base alle specifiche.|
|Vedere quali il framework e la creazione guidata applicazione MFC offrono senza l'aggiunta di una riga di codice personalizzato.|Compilare lo scheletro dell'applicazione ed eseguirla in Visual C++.|L'applicazione in esecuzione lo scheletro deriva standard molte **File**, **modificare**, **visualizzazione**, e **Guida** i comandi di menu da framework. Per le applicazioni MDI, otterrai anche un menu di Windows completamente funzionale e il framework gestisce la creazione, la disposizione e la distruzione di finestre figlio MDI.|
|Costruire l'interfaccia utente dell'applicazione.|Usare Visual C++ [editor di risorse](../windows/resource-editors.md) per modificare visivamente l'interfaccia utente dell'applicazione:<br /><br /> -Creazione rapida.<br />-Definire i tasti di scelta rapida.<br />: Consente di creare finestre di dialogo.<br />-Creare e modificare le bitmap, icone e cursori.<br />: Consente di modificare la barra degli strumenti creata automaticamente dalla creazione guidata applicazione MFC.<br />-Creare e modificare le altre risorse.<br /><br /> È anche possibile testare le finestre di dialogo nell'editor finestre.|Il file di risorse predefinito creato per la creazione guidata applicazione MFC fornisce molte delle risorse che necessarie. Visual C++ è possibile modificare le risorse esistenti e aggiungere nuove risorse in modo semplice e visivamente.|
|Eseguire il mapping di menu per le funzioni del gestore.|Usare il **eventi** pulsante il [finestra proprietà](/visualstudio/ide/reference/properties-window) connettersi i menu e tasti di scelta rapida per le funzioni del gestore nel codice.|Finestra proprietà consente di inserire le voci della mappa messaggi e i modelli di funzione vuota nei file di origine si specificano e si gestisce numerose attività di codifica manuale.|
|Scrivere il codice del gestore.|Utilizzare Visualizzazione classi per passare direttamente al codice nell'editor del codice sorgente. Compilare il codice per le funzioni del gestore. Per altre informazioni sull'utilizzo di visualizzazione classi e sulle procedure guidate che aggiungere codice a un progetto, vedere [aggiunta di funzionalità con creazioni guidate codice](../ide/adding-functionality-with-code-wizards-cpp.md).|Visualizzazione classi consente di aprire l'editor, scorre per il modello di funzione vuota e posiziona il cursore per l'utente.|
|Eseguire il mapping ai comandi di pulsanti della barra degli strumenti.|Eseguire il mapping di ogni pulsante sulla barra degli strumenti a un comando di menu di scelta o tasto di scelta rapida assegnando il pulsante, l'ID del comando appropriato.|Il framework controlla il disegno, l'abilitazione, disabilitazione, controllo e altri aspetti visivi dei pulsanti della barra degli strumenti.|
|Testare le funzioni del gestore.|Ricompilare il programma e usare gli strumenti di debug integrati per testare il corretto funzionamento dei gestori.|È possibile eseguire o tramite il codice per vedere come vengono chiamati i gestori di eventi di traccia. Se è stato compilato il codice, i gestori di eseguono i comandi. Il framework verrà disabilitato automaticamente le voci di menu e pulsanti della barra degli strumenti che non sono gestiti.|
|Aggiungere [finestre di dialogo](../mfc/dialog-boxes.md).|Con l'editor finestre di progettazione risorse del modello di finestra di dialogo. Creare quindi una classe di finestra di dialogo e il codice che gestisce la finestra di dialogo.|Il framework gestisce la finestra di dialogo e semplifica il recupero delle informazioni immesse dall'utente.|
|Inizializzare, convalidare e recuperare i dati finestra di dialogo.|È anche possibile definire come i controlli della finestra di dialogo devono essere inizializzata e convalidata. Usare Visual Studio per aggiungere variabili membro alla classe di finestra di dialogo e associarli ai controlli finestra di dialogo. Specificare le regole di convalida da applicare a ogni controllo quando l'utente ne immette di dati. Fornire il proprio convalide personalizzate se si desidera.|Il framework gestisce la convalida e l'inizializzazione della finestra di dialogo. Se l'utente immette informazioni non valide, il framework visualizza una finestra di messaggio e consente all'utente di immettere nuovamente i dati.|
|Creazione di classi aggiuntive.|Usare Visualizzazione classi per creare documenti aggiuntive, visualizzazione e classi frame-window oltre a quelli creati automaticamente dalla creazione guidata applicazione MFC. È possibile creare classi di recordset aggiuntiva del database, le classi di finestra di dialogo e così via. (Con Visualizzazione classi, è possibile creare classi non derivate da classi MFC.)|Visualizzazione classi aggiunge queste classi per i file di origine e consente di definire le connessioni a tutti i comandi che gestiscono.|
|Aggiungere componenti pronti da usare per l'applicazione.|Usare il `New Item dialog box` per aggiungere un'ampia gamma di elementi.|Questi elementi sono facili da integrare nell'applicazione e di risparmiare una gran quantità di lavoro.|
|Implementare la classe del documento.|Implementare la classe documento specifico dell'applicazione o le classi. Aggiungere variabili membro per contenere le strutture di dati. Aggiungere le funzioni membro per fornire un'interfaccia ai dati.|Il framework sappia già come interagire con i file di dati documento. È possibile aprire e chiudere i file di documento, leggere e scrivere i dati del documento e altre interfacce utente di gestire. È possibile concentrarsi sul modo in cui vengono modificati i dati del documento.|
|Implementare apertura, salvataggio e salvare come comandi.|Scrivere codice per il documento `Serialize` funzione membro.|Il framework Visualizza le finestre di dialogo per il **Open**, **salvare**, e **Salva con nome** comandi il **File** menu. Scrive e legge nuovamente un documento usando il formato dati specificato nella `Serialize` funzione membro.|
|Implementare una classe di visualizzazione.|Implementare uno o più classi di visualizzazione corrispondenti ai documenti. Implementare funzioni membro della vista che è stata mappata all'interfaccia utente con Visualizzazione classi. Svariati [CView](../mfc/reference/cview-class.md)-sono disponibili, tra cui le classi derivate [CListView](../mfc/reference/clistview-class.md) e [CTreeView](../mfc/reference/ctreeview-class.md).|Il framework gestisce la maggior parte della relazione tra un documento e la relativa visualizzazione. Funzioni membro della vista accedono per eseguire il rendering propria immagine nella schermata o pagina stampata e aggiornare le strutture di dati del documento in risposta all'utente i comandi di modifica dei documenti della visualizzazione.|
|Migliora la stampa predefinita.|Se è necessario il supporto della stampa a più pagine, eseguire l'override di funzioni membro di visualizzazione.|Il framework supporta il **Print**, **Imposta pagina**, e **anteprima di stampa** comandi il **File** menu. È necessario indicare la modalità di suddivisione del documento in più pagine.|
|Aggiungere lo scorrimento.|Se è necessario supportare lo scorrimento, derivare la classe o Visualizzazione classi [CScrollView](../mfc/reference/cscrollview-class.md).|La vista aggiunge automaticamente le barre di scorrimento quando la finestra di visualizzazione diventa troppo piccola.|
|Creare le visualizzazioni di form.|Se si desidera basare le visualizzazioni su risorse del modello di finestra di dialogo, derivare la classe o Visualizzazione classi [CFormView](../mfc/reference/cformview-class.md).|La vista utilizza la risorsa modello di finestra di dialogo per visualizzare i controlli. L'utente può spostarsi dal controllo del codice al controllo nella visualizzazione.|
|Creare moduli di database.|Se si desidera che un'applicazione di accesso ai dati basato su form, derivare la classe di visualizzazione da [CRecordView](../mfc/reference/crecordview-class.md) (per la programmazione con ODBC).|La vista funziona come una visualizzazione di form, ma i relativi controlli sono connessi ai campi di un [CRecordset](../mfc/reference/crecordset-class.md) oggetto che rappresenta una tabella di database. MFC Sposta i dati tra i controlli e set di record per l'utente.|
|Creare un semplice testo dell'editor.|Se si desidera la visualizzazione deve essere un semplice editor di testo, derivare la classe o Visualizzazione classi [CEditView](../mfc/reference/ceditview-class.md) oppure [CRichEditView](../mfc/reference/cricheditview-class.md).|La vista fornisce la modifica funzioni, supporto per gli Appunti e file di input/output. `CRichEditView` fornisce il testo con stile.|
|Aggiungere finestre con separatore.|Se si desidera supportare la suddivisione di finestra, aggiungere un [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) collegherà della finestra e oggetto per la finestra cornice SDI o una finestra figlio MDI [OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient) funzione membro.|Il framework fornisce controlli barra di divisione-finestra accanto alle barre di scorrimento e gestisce la suddivisione della visualizzazione in più riquadri. Se l'utente si divide una finestra, il framework crea e associa gli oggetti di visualizzazione aggiuntive al documento.|
|Compilare, testare e il debug dell'applicazione.|Usare le funzionalità di Visual C++ per compilare, testare e il debug dell'applicazione.|Visual C++ è possibile impostare altre opzioni di compilazione e collegamento. Consente inoltre di passare la struttura di classe e codice sorgente.|
